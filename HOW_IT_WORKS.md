## What Happens

### 1. Players Join
- Everyone scans QR code
- Enters their name
- System randomly assigns them to Horse A, B, or C

### 2. Start Race
- Click "Start Race" button
- 3... 2... 1... countdown

### 3. Staggered Start
```
Time 0s:  Gate opens for Horse A ğŸ´ - Team A starts tapping!
Time 2s:  Gate opens for Horse B ğŸ - Team B starts tapping!
Time 4s:  Gate opens for Horse C ğŸ¦„ - Team C starts tapping!
```

### 4. TAP TO WIN!
- **Every player has a GIANT tap button on their phone**
- **All taps from the same team make the same horse go faster**
- **More players tapping = faster horse**
- **Stop tapping = horse slows down (friction)**

### 5. First to Finish Wins
- Winner is determined by fastest time from THEIR start
- So Horse C can still win even though they started last!

## Why Staggered Start?

This teaches the **Transaction Priority** concept:
- Some transactions (Horse A) get processed immediately
- Others (Horse B, C) have to wait their turn
- But with enough processing power (tapping), later transactions can still finish fast!
